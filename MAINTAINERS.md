# Maintainers

Tax AI is maintained as an open source framework for tax-focused human-AI collaboration.

## Project Owner

- [@panghim](https://github.com/panghim): project owner, product direction, repository administration and release approval.

## Maintainers

- [@loyputh](https://github.com/loyputh): invited maintainer with write access for implementation work, bug fixes, refactors and pull request collaboration.

## AI-Assisted Maintenance

- Codex: repository-level structure, framework contracts, verification, issue queue grooming and PR readiness.
- Cursor: focused local implementation, component cleanup and scoped refactors.

## Contribution Expectations

- Use issues for non-trivial work before opening large pull requests.
- Keep framework-facing changes aligned with `docs/framework-architecture.md`.
- Run `npm run check` before marking a pull request ready.
- Do not present tax calculations, declarations or AI advice as production-ready unless source, jurisdiction and date metadata are included.
- Keep unrelated local projects and repositories out of Tax AI changes.

## Access Policy

New maintainers should start with `write` access. Elevate to `maintain` only after sustained contribution and agreement from the project owner.
